# Nexor Architecture

## Overview

Nexor is designed as a modular Manufacturing Execution System (MES) focused on textile print production.

The system processes production data generated by printing machines and transforms it into structured operational information.

---

# Data Sources

Primary data source:

Machine production logs

Example:

[General]
ComputerName = DESKTOP-2GGH09O
SoftwareVersion = 10.0.17
JobID = 10864
Document = file.jpeg
StartTime = 03/03/2026 23:19:28
EndTime = 03/03/2026 23:22:07
Driver = Mimaki TS55-1800

---

# Log Processing

The logs module is responsible for:

* parsing raw logs
* extracting production metadata
* calculating production duration
* identifying machines
* normalizing records

---

# Machine Identification

Machines are identified using the field:

ComputerName

Example configuration:

Machine: M2
ComputerName: DESKTOP-2GGH09O
Model: Mimaki TS55-1800

---

# Production Model

Core data entity:

ProductionJob

Fields include:

* job_id
* machine
* document
* start_time
* end_time
* duration
* fabric
* length
* gap_before

---

# Gap Handling

Field `VPosMM` represents the advance before printing.

In Nexor it may be represented as:

SPACE

This allows the system to track:

* technical gaps
* fabric changes
* roll transitions
* production waste

---

# Future Architecture

The platform is designed to evolve to:

Cloud platform + Local agent

The Nexor Agent will:

* monitor log folders
* automatically import logs
* normalize production data
* synchronize with the platform

This architecture allows operation even when internet connectivity is unstable.
